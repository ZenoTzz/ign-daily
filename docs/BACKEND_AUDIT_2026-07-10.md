# IGN Daily 后端审查报告（2026-07-10）

## 审查边界

本次审查覆盖私有 FastAPI、SQLite 鉴权与任务队列、RSS/翻译脚本、Google Docs
同步、GitHub Actions、服务器安装/部署/备份/恢复脚本，以及线上只读运行状态。

修复只涉及代码、测试和运维配置。仓库中的 `data/`、现有译文、润色稿和 Google
Docs 正文均未被批量改写。线上仅新增了一份备份，并将两个 `.env` 的权限收紧为
`600`。

## 当前数据流

1. `igndaily.site` 的 `/srv/ign-daily/data` 是服务器运行时数据源。
2. FastAPI 负责登录、文章读取、JSON 写回、翻译请求和 Codex 任务状态。
3. GitHub 保存代码和已提交的内容快照；静态部署必须保留服务器运行时数据。
4. 完成的译文再由桌面端 OAuth 凭据增量同步到 Google Docs。

服务器定时任务设置了 `IGN_DAILY_SKIP_GIT=1`，因此“服务器新增内容进入 GitHub”
不是一个完全自动的通用复制过程，仍依赖现有 Codex/人工提交工作流。不要把服务器
仓库的 `git status` 当成运行时数据是否安全的判断依据。

## 已修复问题

### 严重

- 静态部署只更新 `/srv/ign-daily/server_api/ign_daily_api.py`，运行中的
  `/srv/ign-daily-api/ign_daily_api.py` 长期不更新。部署现会更新独立 API 副本、按需
  安装依赖、重启、健康检查，并在失败时回滚 API 模块。
- 通用 `/files/{path}` 接口原本允许登录令牌读取 `.env` 或改写 Python/HTML。现在只
  允许访问 `data/` 下的 JSON，并阻止路径穿越、隐藏路径和软链接逃逸。
- 部署末尾对全站执行 `chmod 644`，会把包含 GitHub PAT 和翻译密钥的主 `.env` 改成
  全局可读。部署和配置脚本现在强制两个 `.env` 为 `600`。

### 高

- `.env` 在模块常量计算之后才加载，导致存储模式、CORS、仓库路径等配置可能被
  静默忽略。现已在派生所有运行配置前加载，并由 systemd 同时声明 EnvironmentFile。
- `with db()` 原本只提交事务、不关闭 SQLite 连接。现已改为真正的上下文管理器，
  所有连接都在 `finally` 中关闭，并设置 15 秒 busy timeout。
- 翻译后处理会先把文章标记为 `done`，最后才返回校验失败。现在任何标题、摘要、
  副标题、段落或标点校验失败时都不会改写译文、索引或历史统计。
- Codex 完成接口可在译文文件不存在时把任务标为完成。现在会返回 409，并列出缺失
  的文章 ID。
- 本地 JSON 写入不是原子操作，也没有使用前端已经声明的版本字段。现在采用临时
  文件加 `os.replace`，并以内容 SHA 拒绝陈旧的更新或删除。
- 线上最后一份完整备份停在 6 月 24 日；直接复制活动中的 SQLite 也不保证一致。
  备份现使用 SQLite backup API、临时归档和失败清理，安装脚本增加每日 03:35 备份。
- 恢复脚本会直接删除当前 `data/`，且不检查归档路径。现在恢复前自动备份，拒绝
  绝对路径、`..`、软/硬链接成员，并用分阶段目录替换运行时数据。

### 中

- 登录接口没有失败次数限制，且不存在的用户会更快返回。现在无论用户是否存在都
  执行同等 PBKDF2 校验，并按客户端地址限制 15 分钟内最多 8 次失败。
- Google Docs 同步写死 `D:\Daily News` 凭据路径。同步与导入脚本现在同时支持配置
  文件、环境变量和相对仓库路径，刷新令牌按私密权限写入；缺少 OAuth 客户端文件时
  不再先删除旧令牌。
- `git_push.py` 曾把 PAT 放进远程 URL，可能出现在进程列表或日志。现在用临时 Git
  HTTP 请求头，并在 pull 失败时停止 push。
- 安装脚本曾递归 `chown /srv`，并会删除非 Git 的目标目录。现在只管理明确目录，
  校验路径必须位于 `/srv/`，拒绝替换未知的非空目录。
- 静态部署过去没有保留 `ign_rss_new.json`。现在它与 `data/`、汇率文件一起保留。

## 线上核对摘要

- Nginx 和 FastAPI 均为 active，API 仅监听 `127.0.0.1:8010`，公网由 Nginx 代理。
- 服务器运行时 API 与站点仓库内 API 的哈希不同，证实独立 API 副本未随部署更新。

  本地运行时数据并行造成的状态，不应通过强制 reset 修复。
- 审查时数据库包含 1 个用户、3 个会话记录和 10 个任务。
- 已创建 `/srv/ign-daily-backups/ign-daily-20260710-140435.tar.gz` 并验证可列出内容。

## 保留的架构取舍

- 多文件业务操作（如恢复过滤文章）仍不是数据库事务；单文件写入已原子化，但极端
  的磁盘/网络故障仍可能留下部分成功状态。若以后改为多人并发编辑，应把文章元数据
  迁入数据库或增加正式的事务日志。
- Google Docs 使用桌面 OAuth，适合当前人工确认工作流，但不适合作为无人值守的
  服务器级灾备。Google Docs 应继续作为下游副本，不应成为唯一数据源。
- 服务器运行时内容与 GitHub 内容是双源。当前优先保护服务器数据，没有自动用
  GitHub 覆盖 `data/`。若要实现严格双向复制，需要先定义冲突优先级，不能简单执行
  `git reset --hard` 或部署时覆盖整个 `data/`。
